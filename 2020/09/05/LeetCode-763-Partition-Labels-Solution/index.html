LeetCode-763-Partition-Labels-Solution

Posted on 2020-09-05 In 刷题解析 Views:

Problem Description: A string S of lowercase English letters is given. We want to partition this string into as many parts as possible so that each letter appears in at most one part, and return a list of integers representing the size of these parts.

Difficulty: Medium

Example 1:

S = “ababcbacadefegdehijhklij”
Output: [9, 7, 8]
Reasoning: In the first section which contains the first 9 characters, characters a, b, c, d only show up in this section. Similarly, in the second section, characters d, e, f, g are present and in the last section characters h, i, j, k, l are detected only in this part.

Algorithm: Since each kind of character is only allowed to show up in one part, it is natural to consider mapping the start and ending positions of each character, i.e. for each type of character we have to know the indices where it first and last appears. It is guaranteed that such a region has to be covered by a certain section in our final answer. In this context, such question has been transformed into such a question: given the several intervals, how can we make some splits so that each interval only existed in one section and the number of section should be maximized? This is typical greedy problem and we can be inspired by similar problems like: LeetCode 435 Non-overlappping intervals or LeetCode 452 Minimum Number of Arrows to Burst Balloons. In this algorithm, we first build a data structure to record the starting and ending indices for each character (array, hashmap, whatever). Then we sort the distribution according to the starting indices. Next, we look at each interval, and meanwhile use two variables to record the current section. Specifically, when the current interval being looked at has its starting indices larger than the current right, we know it is the end of the current section and add it to the final list and update the right location. If not, we check the ending indices, and if it is larger than the right position, we simply update the right border. Here goes the code. In this algorithm, several passes all adopt a time complexity of O(n), and sorting takes O(nlogn), so the overall complexity if O(nlogn). Although we use extra array to record the indices, but since there are at most 26, the space complexity if O(1).

class Solution {
    public List<Integer> partitionLabels(String S) {
        int[][] frag=new int[26][2];
        for (int i=0;i<26;i++) {
            frag[i][0]=Integer.MAX_VALUE;
            frag[i][1]=Integer.MAX_VALUE;
        }
        for (int i=0;i<S.length();i++) {
            char c=S.charAt(i);
            if (frag[c-'a'][0]==Integer.MAX_VALUE) {
                frag[c-'a'][0]=i;
                frag[c-'a'][1]=i;
            }
            else {
                frag[c-'a']=i;
            }
        }
        Arrays.sort(frag, new Comparator<int[]>() {
            public int compare(int[] i1, int[] i2) {
                return i1[0]-i2[0];
            }
        });
        int left=0;
        int right=frag[0][1];
        List<Integer> ans=new ArrayList<>();
        for (int i=0;i<26&&frag[i][0]!=Integer.MAX_VALUE;i++) {
            if (frag[i][0]>right) {
                ans.add(right-left+1);
                left=frag[i][0];
                right=frag[i][1];
            }
            if (frag[i][1]>right) right=frag[i][1];
        }
        ans.add(right-left+1);
        return ans;
    }
}

If we take a closer look at the algorithm above, you can notice several unnecessities. For example, the sorting processes, which extends the time complexity from O(n) to O(nlogn), is actually unnecessary. The purpose of this step is simply to guarantee that we look at the string from left to right, which could be realized by iterating the string from left to right. Out of this consideration, we can avoid the recording of the starting indices but only focus on the ending indices. In terms of the update of the left and right border, the left border will be updated only when the pointer we are iterating with goes beyond the current right border. The right border will be updated in two cases. First, if the current character has a further ending index than the current right border. Second, the left border is updated. In this algorithm, the time complexity is compressed to O(n). Here goes the code.

class Solution {
    public List<Integer> partitionLabels(String S) {
        HashMap<Character, Integer> map=new HashMap<>();
        for (int i=0;i<S.length();i++) {
            map.put(S.charAt(i), i);
        }
        List<Integer> ans=new ArrayList<>();
        int left=0;
        while (left<S.length()) {
            int right=left;
            for (int i=left;i<=right;i++) {
                int ending=map.get(S.charAt(i));
                if (ending>right) {
                    right=ending;
                }
            }
            ans.add(right-left+1);
            left=right+1;
        }
        return ans;
    }
}
